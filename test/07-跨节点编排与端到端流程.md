# 测试用例 · 跨节点编排与端到端流程

> 模块：Orchestrator 主 Agent · dispatch（跨节点编排）+ 全链路
> 前缀：`FLOW`
> 覆盖验收点：架构.md §4（四类核心数据流）+ §4.4 混合意图流 + 原则 P3、P5

---

## 验收目标

1. **§4.1 知识问答流**：意图→Knowledge→质量判断→（重检）→直出/坦诚告知。
2. **§4.2 购物执行流**：意图→Shopping 编排→转述用户。
3. **§4.3 举手流**：子节点返回 NEED_CLARIFY→主 Agent 开口。
4. **§4.4 混合意图流**：主 Agent 串联 Knowledge + Shopping，子节点彼此不通信。
5. 每步结果回主 Agent，记忆始终单点。
6. 子节点在整个流程中无状态、不开口。

---

## FLOW-001 知识问答端到端（含一次重检）（正向）★

- 模块：跨节点·知识问答流
- 用例类型：正向
- 优先级：高
- 前置条件：用户"小米14影像规格怎么样"
- 测试步骤：
  1. 主 Agent 意图识别 → KNOWLEDGE
  2. 注入快照委派 Knowledge
  3. 第1轮质量判断 INSUFFICIENT → 重检
  4. 第2轮换策略 → SUFFICIENT
  5. 主 Agent 直出答案
- 预期结果：
  - 链路：User → Orchestrator → Knowledge → (judge loop) → Orchestrator → User
  - 共 2 次 Knowledge 调用
  - 最终答案由主 Agent 生成并开口（P2）
  - Knowledge 全程不直接回应用户
- 需求来源：架构.md §4.1

---

## FLOW-002 购物执行端到端（正向）

- 模块：跨节点·购物执行流
- 用例类型：正向
- 优先级：高
- 前置条件：用户"帮我加购一台小米14 16+512"
- 测试步骤：
  1. 主 Agent 意图识别 → TOOL（加购，槽位已齐）
  2. 委派 Shopping 编排 add_cart
  3. Shopping 返回 SUCCESS + cartId
  4. 主 Agent 转述给用户
- 预期结果：
  - 链路：User → Orchestrator → Shopping → MCP工具 → Orchestrator → User
  - 主 Agent 收结果后转述（不原样转发）
  - 用户收到加购成功反馈
- 需求来源：架构.md §4.2

---

## FLOW-003 举手流端到端：缺槽位→主 Agent 澄清（正向）★ P4

- 模块：跨节点·举手流
- 用例类型：正向
- 优先级：高
- 前置条件：用户"帮我下单小米14"，缺收货地址
- 测试步骤：
  1. 主 Agent 识别意图=下单，槽位检查发现缺地址（或委派后 Shopping 发现）
  2. 返回 NEED_CLARIFY + missingSlots=["收货地址"]
  3. 主 Agent 开口反问
- 预期结果：
  - 子节点（Shopping）不直接问用户
  - 开口权在主 Agent（P2/P4）
  - 用户补充地址后，主 Agent 重新注入完整槽位继续流程
- 需求来源：架构.md §4.3；原则 P4

---

## FLOW-004 混合意图流：主 Agent 串联两个子节点（正向）★ P5

- 模块：跨节点·混合意图流
- 用例类型：正向
- 优先级：高
- 前置条件：用户"帮我推荐一款适合打游戏的手机，合适就直接加购"
- 测试步骤：
  1. 主 Agent 识别混合意图（检索 + 外部调用）
  2. ①先委派 Knowledge 推荐 → 返回候选机型 + 信号
  3. 主 Agent 质量判断充分后转述、引导确认
  4. 用户确认某机型
  5. ②委派 Shopping 执行加购（确认的机型）
  6. 主 Agent 转述结果
- 预期结果：
  - **串联顺序由主 Agent 决定**（先检索确认、再执行动作）
  - Knowledge 与 Shopping **彼此不直接通信**（P5，各自无状态）
  - 每步结果回主 Agent，由主 Agent 判断是否进下一步
  - 记忆始终单点（P3）
- 需求来源：架构.md §4.4
