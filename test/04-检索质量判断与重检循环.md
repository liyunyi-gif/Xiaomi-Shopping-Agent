# 测试用例 · 检索质量判断与重检循环

> 模块：Orchestrator 主 Agent · judge（★ 纯自研，简历重点）
> 前缀：`JUDGE`
> 覆盖验收点：架构.md §5（三信号4步判定 + 重检状态机）+ 原则 P6（质量判断上移）、P7（克制原则）

---

## 验收目标

1. **三信号4步组合判定**（优先级固定，可执行）：
   1. 召回数=0 → FAILED（失败）
   2. 召回非0，关键实体未命中 → INCOMPLETE（不全）
   3. 实体命中，相关度<阈值 → INSUFFICIENT（不够）
   4. 三者满足 → SUFFICIENT（充分）
2. **重检循环状态机**：不够则重检，上限 N=2（含首查共 3 次调用）。
3. **换策略重检**：每轮必须换策略（换词/拆子问题），禁止原样重查。
4. **达上限退化坦诚告知**：绝不硬答（防幻觉）。
5. 判断基于可量化信号，不靠"感觉"。

## JUDGE-001 召回数为0 → 判 FAILED（正向/第1步）

- 模块：质量判断·4步判定
- 用例类型：正向
- 优先级：高
- 前置条件：Knowledge 返回 recallCount=0
- 测试步骤：
  1. 主 Agent 质量判断
- 预期结果：
  - QualityVerdict.level = FAILED
  - 直接进入重检或（已达上限）坦诚告知
- 需求来源：架构.md §5.1（优先级1）

- 优先级：高

- 需求来源：架构.md §5.1（优先级2）

## JUDGE-004 三信号满足 → 判 SUFFICIENT 直出（正向/第4步）

- 模块：质量判断·4步判定
- 用例类型：正向
- 优先级：高
- 前置条件：recallCount=5，实体命中，topScore=0.85 ≥ 阈值0.7

- 需求来源：架构.md §5.1（优先级4）

## JUDGE-005 重检循环：不够→换策略重检→充分（正向）★

- 模块：质量判断·重检状态机
- 用例类型：正向
- 优先级：高
- 前置条件：第1轮判 INSUFFICIENT，retryAttempt=0 < maxAttempts=2
- 测试步骤：
  1. 第1轮不够 → 进入重检
  2. 第2轮换策略重写后召回
  3. 第2轮判 SUFFICIENT
- 预期结果：
  - 共发起 2 次检索调用
  - 第2轮 retryAttempt=1
  - 第2轮使用不同重写策略（非原样）
  - 最终 SUFFICIENT 直出
- 需求来源：架构.md §5.2（重检循环状态机）

---

## JUDGE-006 重检达上限仍不够 → 坦诚告知（边界）★ P7

- 模块：质量判断·克制原则
- 用例类型：边界
- 优先级：高
- 前置条件：retryAttempt 已达 maxAttempts=2，仍判 INSUFFICIENT
- 测试步骤：
  1. 检查重检次数
  2. 达上限，不再重检
- 预期结果：
  - 不再发起第 3 次检索（含首查共 3 次上限）
  - 退化为「坦诚告知 + 引导澄清」
  - **绝不硬编答案**（防幻觉）
- 需求来源：架构.md §5.2；原则 P7

---

## JUDGE-007 重检必须换策略，禁止原样重查（约束）★ P7

- 模块：质量判断·换策略
- 用例类型：正向
- 优先级：高
- 前置条件：进入重检
- 测试步骤：
  1. 记录第1轮使用的策略（如 原始query）
  2. 触发重检
  3. 检查第2轮传入 KnowledgeRequest 的 query/策略
- 预期结果：
  - 第2轮 query ≠ 第1轮 query（已重写：换关键词/拆子问题）
  - retryAttempt 递增
  - 禁止用相同 query 原样发起第二次检索
- 需求来源：架构.md §5.2（禁止原样重查）；原则 P7

---

## JUDGE-008 质量判断基于可量化信号，不靠主观感觉（约束）★ P6

- 模块：质量判断·信号驱动
- 用例类型：正向
- 优先级：中
- 前置条件：任意质量判断
- 测试步骤：
  1. 审查 QualityVerdict 生成逻辑
- 预期结果：
  - level 由 topScore / hitEntities / recallCount 三信号 + 阈值规则推导
  - reason 字段记录触发信号的具体数值
  - 不存在 LLM 主观打"我觉得好不好"的判断路径
- 需求来源：架构.md §5.1（规则初筛）；原则 P6
