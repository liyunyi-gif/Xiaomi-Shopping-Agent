# 测试用例 · 三层记忆（★ 简历重点）

> 模块：Orchestrator 主 Agent · memory（①短期Redis / ②长期PgSQL / ③存档PgSQL）
> 前缀：`MEM`
> 覆盖验收点：架构.md §6（三层记忆模型 + 加载注入 + 写入遗忘）+ 原则 P3（记忆单点）

---

## 验收目标

1. **对话存档 ≠ 记忆**：逐字历史与提炼状态分层，不混存。
2. ①短期记忆：Redis，最近 N 轮原文 + 当前意图/槽位，会话级。
3. ②长期记忆：PgSQL 提炼（画像/偏好/决策点/槽位），跨会话。
4. ③对话存档：PgSQL 逐字实时落盘，**不进默认上下文**。
5. 加载注入 = ②长期 + ①最近N轮；存档③仅按需检索召回。
6. 写入：会话结束批量提炼入②；存档③逐字实时。
7. 遗忘：过期/低权重淘汰 + 用户手动清除。
8. **子节点零记忆**（P3）：每次调用快照由主 Agent 注入，调用结束即释放。

---

## MEM-001 对话存档与记忆分层，不混存（约束）

- 模块：三层记忆·分层
- 用例类型：正向
- 优先级：高
- 前置条件：一次完整会话
- 测试步骤：
  1. 检查存档落点 vs 记忆落点
- 预期结果：
  - 逐字历史 → t_message（存档③）
  - 提炼状态 → t_user_longterm_memory（记忆②）
  - 两者物理分离，存档表无提炼字段，记忆表无逐字内容
  - 存档③ **不被注入默认上下文**
- 需求来源：架构.md §6.1（红线）、§6.2

---

## MEM-002 子节点零记忆，快照由主 Agent 注入释放（约束）★ P3

- 模块：三层记忆·记忆单点
- 用例类型：正向
- 优先级：高
- 前置条件：连续两次调用 Knowledge
- 测试步骤：
  1. 调用1：注入 snapshot_A，调用结束
  2. 调用2：注入 snapshot_B
  3. 检查 Knowledge 内部是否残留 snapshot_A
- 预期结果：
  - Knowledge 不持有任何跨调用的状态字段
  - 调用2 仅能看到 snapshot_B，无 snapshot_A 残留
  - 记忆只存在于主 Agent（P3）
- 需求来源：架构.md §6.5；原则 P3

## MEM-003 短期记忆① Redis 存最近N轮（正向）

- 模块：三层记忆·短期
- 用例类型：正向
- 优先级：高
- 前置条件：配置 short-term-window=10；进行 12 轮对话
- 测试步骤：
  1. 查 Redis key `chat:session:{sid}:messages`
- 预期结果：
  - List 仅保留最近 10 轮（老的滚动淘汰）
  - 含 User/Assistant 消息对
  - TTL 为会话级（如 24h）
- 需求来源：架构.md §6.2（①）；技术架构总纲 §5.2

## MEM-004 长期记忆② 跨会话注入（正向）★

- 模块：三层记忆·长期
- 用例类型：正向
- 优先级：高
- 前置条件：用户A 曾表达偏好"预算5000以内，爱打游戏"已提炼入②
- 测试步骤：
  1. 用户A 开启新会话
  2. 主 Agent 加载记忆
- 预期结果：
  - 新会话默认上下文含长期记忆②（偏好"预算5000/打游戏"）
  - 新会话无需用户重述即可被推荐利用
  - 实现"跨会话记得你"
- 需求来源：架构.md §6.2（②）、§6.3

---

## MEM-005 会话结束批量提炼入长期记忆（正向）★

- 模块：三层记忆·提炼写入
- 用例类型：正向
- 优先级：高
- 前置条件：一次会话中用户多次提及偏好/决策（"想要大内存""定了小米14"）
- 测试步骤：
  1. 触发会话结束
  2. 批量提炼（LLM）
  3. 写入 t_user_longterm_memory
- 预期结果：
  - 提炼出结构化记忆条目（mem_type=偏好/决策点，weight）
  - 非逐字，是提炼后的小体量状态
  - 提炼在会话结束批量执行，非每轮执行（成本可控）
- 需求来源：架构.md §6.4（写入）

## MEM-006 对话存档③逐字实时落盘且不进默认上下文（正向）

- 模块：三层记忆·存档
- 用例类型：正向
- 优先级：中
- 前置条件：会话进行中，每轮有往返
- 测试步骤：
  1. 每轮对话结束检查 t_message
  2. 检查默认上下文构造
- 预期结果：
  - 每轮 User/Assistant 消息实时写入 t_message（不经提炼）
  - 默认上下文 = ② + ①，**不含③逐字历史**
  - 防止 token 爆炸（存档量大但不入上下文）
- 需求来源：架构.md §6.4（存档③逐字实时）、§6.1（红线）

---

## MEM-007 加载注入=②长期+①最近N轮，存档③仅按需召回（正向）

- 模块：三层记忆·加载注入
- 用例类型：正向
- 优先级：高
- 前置条件：用户打开 Agent
- 测试步骤：
  1. 主 Agent 构造默认上下文
  2. 用户请求查看历史
- 预期结果：
  - 默认上下文 = 长期记忆② + 最近N轮①（轻量、高信噪比）
  - 用户要看历史时，从存档③检索/回显（可复用 Knowledge 检索能力）
  - "记得你"靠②注入，"看历史"靠③回显，两条路径解耦
- 需求来源：架构.md §6.3

---

## MEM-008 遗忘：过期淘汰 + 用户手动清除（异常/边界）

- 模块：三层记忆·遗忘
- 用例类型：边界
- 优先级：中
- 前置条件：②中存在陈旧低权重条目（很早浏览、未转化）+ 用户主动请求清除
- 测试步骤：
  1. 触发过期/低权重淘汰逻辑
  2. 用户调用记忆清除接口
- 预期结果：
  - 陈旧低权重条目被衰减/移除（防止无限膨胀）
  - 用户手动清除后，②对应条目被删除（隐私可控）
  - 清除操作可针对单用户/单会话
- 需求来源：架构.md §6.4（遗忘）
