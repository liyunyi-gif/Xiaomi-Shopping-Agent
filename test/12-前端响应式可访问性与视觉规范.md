## FEUX-001 桌面端三栏布局
- 模块：前端响应式 / 桌面端
- 用例类型：正向
- 优先级：高
- 前置条件：视口宽度 ≥ 1024px
- 测试步骤：
  1. 打开页面。
  2. 查看整体布局。
- 预期结果：
  - 页面呈现左侧推荐/快捷操作、中间聊天、右侧购物车/状态面板三栏。
  - 聊天区域是视觉中心。
  - 无横向滚动。
- 需求来源：前端UIUX设计文档 §4.1；前端技术架构文档 §7.3

## FEUX-002 平板端两栏布局
- 模块：前端响应式 / 平板端
- 用例类型：正向
- 优先级：中
- 前置条件：视口宽度 768px 至 1023px
- 测试步骤：
  1. 调整视口到平板宽度。
  2. 查看布局。
- 预期结果：
  - 页面呈现聊天 + 侧边 Tab/面板的两栏布局。
  - 推荐、购物车、状态信息可在侧边区域切换。
  - 输入框可见且不被遮挡。
- 需求来源：前端UIUX设计文档 §4.2；前端技术架构文档 §7.3

## FEUX-003 移动端单栏聊天优先
- 模块：前端响应式 / 移动端
- 用例类型：正向
- 优先级：高
- 前置条件：视口宽度 ≤ 480px
- 测试步骤：
  1. 调整视口到 375px。
  2. 打开页面。
- 预期结果：
  - 页面单栏展示，聊天区域优先。
  - 快捷操作可横向滚动或紧凑展示。
  - 推荐/购物车/状态通过底部 Sheet 或折叠区访问。
  - 无页面级横向滚动。
- 需求来源：前端UIUX设计文档 §4.3 / §10

## FEUX-004 移动端输入框不被固定元素遮挡
- 模块：前端响应式 / 输入体验
- 用例类型：边界
- 优先级：高
- 前置条件：移动端视口，输入框聚焦
- 测试步骤：
  1. 聚焦输入框。
  2. 模拟软键盘出现或缩小视口高度。
- 预期结果：
  - 输入框仍可见。
  - 发送按钮可触达。
  - 消息列表底部不被固定栏遮挡。
- 需求来源：前端UIUX设计文档 §10.2

## FEUX-005 颜色与文字对比度
- 模块：前端视觉规范 / 无障碍
- 用例类型：边界
- 优先级：高
- 前置条件：页面所有主要状态渲染完成
- 测试步骤：
  1. 检查正文、按钮、状态芯片、错误提示的对比度。
- 预期结果：
  - 正文文本对比度 ≥ 4.5:1。
  - 大文本/图标对比度 ≥ 3:1。
  - 错误/成功/警告状态在浅色背景下可读。
- 需求来源：前端UIUX设计文档 §7.1 / §11.1

## FEUX-006 键盘导航顺序合理
- 模块：前端无障碍 / 键盘
- 用例类型：正向
- 优先级：高
- 前置条件：页面加载完成
- 测试步骤：
  1. 使用 Tab 从页面顶部开始导航。
  2. 依次聚焦快捷操作、聊天输入、发送按钮、状态面板操作。
- 预期结果：
  - Tab 顺序符合视觉顺序。
  - 当前焦点有可见 focus ring。
  - 所有关键操作可通过键盘触发。
- 需求来源：前端UIUX设计文档 §11.1 / §11.2

## FEUX-007 图标按钮具备可访问名称
- 模块：前端无障碍 / 图标
- 用例类型：正向
- 优先级：高
- 前置条件：页面存在图标按钮，如刷新、关闭 Sheet、发送
- 测试步骤：
  1. 检查图标按钮 DOM。
  2. 使用屏幕阅读器或可访问性检查工具。
- 预期结果：
  - 图标按钮存在可见文本、`aria-label` 或等价可访问名称。
  - 不使用 emoji 作为结构性图标。
- 需求来源：前端UIUX设计文档 §11.1；前端技术架构文档 §10.3

## FEUX-008 reduced-motion 下禁用非必要动效
- 模块：前端动效 / 可访问性
- 用例类型：边界
- 优先级：中
- 前置条件：系统设置 `prefers-reduced-motion: reduce`
- 测试步骤：
  1. 打开页面。
  2. 触发消息出现、卡片出现、状态面板展开。
- 预期结果：
  - 非必要动画减少或关闭。
  - 功能状态仍清晰可见。
  - 不依赖动效传达唯一信息。
- 需求来源：前端UIUX设计文档 §11.1 / §11.3

## FEUX-009 动效时长符合 150–300ms
- 模块：前端动效 / 视觉规范
- 用例类型：正向
- 优先级：低
- 前置条件：页面启用正常动效
- 测试步骤：
  1. 触发按钮 hover/press、卡片出现、状态切换。
- 预期结果：
  - 微交互动效控制在 150–300ms。
  - 不出现超过 500ms 的阻塞性装饰动画。
  - 动效不导致布局抖动。
- 需求来源：前端UIUX设计文档 §7.4 / §11.2

## FEUX-010 设计 token 使用一致
- 模块：前端视觉规范 / token
- 用例类型：正向
- 优先级：中
- 前置条件：主要组件已实现
- 测试步骤：
  1. 检查 Button、Card、StatusChip、MessageBubble 样式。
- 预期结果：
  - 颜色、圆角、间距使用统一 token 或 Tailwind 语义类。
  - 不在组件中散落大量随机 hex 值。
  - 小米橙与 AI 紫蓝有明确使用边界。
- 需求来源：前端UIUX设计文档 §7；前端技术架构文档 §7.2

## FEUX-011 商品图片或卡片内容预留空间
- 模块：前端性能体验 / CLS
- 用例类型：边界
- 优先级：中
- 前置条件：推荐卡片异步出现或图片懒加载
- 测试步骤：
  1. 触发推荐卡片加载。
  2. 观察页面布局。
- 预期结果：
  - 卡片/图片区域预留尺寸。
  - 不出现明显布局跳动。
  - loading 使用 skeleton 或占位。
- 需求来源：前端UIUX设计文档 §11.3

## FEUX-012 触控目标尺寸符合要求
- 模块：前端交互 / 触控
- 用例类型：边界
- 优先级：高
- 前置条件：移动端视口
- 测试步骤：
  1. 检查发送按钮、快捷操作、卡片按钮、状态面板按钮。
- 预期结果：
  - 主要触控目标不小于 44×44px。
  - 相邻触控目标间距不小于 8px 或等价安全间距。
- 需求来源：前端UIUX设计文档 §11.2
