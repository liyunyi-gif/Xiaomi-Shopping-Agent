# 测试用例 · 知识检索（双路召回与重排）

> 模块：Knowledge 子节点 · rewrite / recall / rerank
> 前缀：`RECALL`
> 覆盖验收点：架构.md §3.2（查询重写/子问题拆分/双路召回/并行编排/rerank/置信信号）+ P5 能力正交 + P6 质量判断上移

---

## 验收目标

1. 查询重写：指代消解、口语化转规范术语。
2. 双路召回并行：语义路（HNSW）+ 关键词路（ts_rank）并发执行。
3. 加权 rerank：语义/关键词/字段三路加权，Top-1 精准。
4. Knowledge 只返回客观三信号，**不做主观质量自判**（P6）。
5. 子问题拆分：复杂问题拆并行检索后合并。
6. Knowledge 无状态、不面对用户、不做意图识别（P5）。

---

## RECALL-001 双路召回并行执行（正向）★

- 模块：知识检索·双路召回
- 用例类型：正向
- 优先级：高
- 前置条件：知识库已含"小米14 影像"相关切片；query="小米14拍照怎么样"
- 测试步骤：
  1. 注入 KnowledgeRequest（question="小米14拍照怎么样"）
  2. 触发双路召回（CompletableFuture 并行）
  3. 收集两路结果
- 预期结果：
  - 语义路返回 Top-K 语义相关切片
  - 关键词路返回 Top-K 含关键词切片
  - 两路并行，总耗时 ≈ max(单路)，非 sum（验证并行）
- 需求来源：架构.md §3.2；技术架构 知识库Agent §4

## RECALL-002 查询重写·指代消解与口语化转术语（正向）

- 模块：知识检索·查询重写
- 用例类型：正向
- 优先级：高
- 前置条件：快照 recentContext 中出现"小米14"；当前 query="它的续航多久"
- 测试步骤：
  1. 查询重写读取会话快照
  2. 重写 query
- 预期结果：
  - 指代"它"被消解为"小米14"
  - "续航多久"可规范化为"电池容量/续航时间"
  - 重写后 query 召回质量优于原始 query
- 需求来源：架构.md §3.2（查询重写）

## RECALL-003 子问题拆分并行检索后合并（正向）

- 模块：知识检索·子问题拆分
- 用例类型：正向
- 优先级：中
- 前置条件：query="小米14的影像和续航对比 iPhone15"（多子问题）
- 测试步骤：
  1. 拆分为子问题（影像/续航/对比）
  2. 各子问题并行检索
  3. 合并去重
- 预期结果：
  - 拆出 ≥2 个子问题
  - 各子问题独立检索后结果合并、去重
  - 合并结果覆盖原问题各子维度
- 需求来源：架构.md §3.2（子问题拆分 + 并行编排）

## RECALL-005 外部 rerank 模型重排提升 Top-1 精准（正向）★

- 模块：知识检索·rerank（外部 Qwen3-Reranker-8B）
- 用例类型：正向
- 优先级：高
- 前置条件：双路召回合并去重后 N 条候选；外部 rerank 可用（硅基流动 key 已配）
- 测试步骤：
  1. 双路结果合并去重
  2. 调用 Qwen3-Reranker-8B（POST /v1/rerank，body 含 model/query/documents/top_n）
  3. 按返回的 relevance_score 降序映射回 ScoredDoc，取 Top-1
- 预期结果：
  - 外部 rerank 返回每条候选的 relevance_score（0~1）
  - Top-1 为与 query 最相关的候选（cross-encoder 精度高于双路原始排序）
  - finalScore = relevance_score
  - 最终返回结果列表按 relevance_score 降序
- 需求来源：架构.md §3.2（rerank）；技术架构 知识库Agent §5.2

## RECALL-005b 外部 rerank 失败 → 自动降级自研加权（异常）★

- 模块：知识检索·rerank（Reranker 门面降级）
- 用例类型：异常
- 优先级：高
- 前置条件：外部 rerank 不可用（限流/超时/key 缺失/网络异常）；降级开关开启
- 测试步骤：
  1. 触发外部 rerank，调用失败
  2. Reranker 门面捕获异常
  3. 回退到 WeightedReranker 自研加权打分
- 预期结果：
  - 不向调用方抛异常，流程不中断
  - 改用加权公式 finalScore = 0.5*sim + 0.3*kw + 0.2*field 计算
  - title/spec_text 命中的切片获得字段加权加成
  - 仍返回降序结果列表（精度略低但可用）
- 需求来源：架构.md §3.2（rerank）；技术架构 知识库Agent §5.1/§5.3

## RECALL-006 Knowledge 只返回客观信号，不含主观质量自评（约束）★ P6

- 模块：知识检索·置信信号
- 用例类型：正向
- 优先级：高
- 前置条件：任意检索调用
- 测试步骤：
  1. 检查 KnowledgeResponse 字段
- 预期结果：
  - 包含：results / topScore / hitEntities / recallCount（客观可量化）
  - **不含**任何"isGood"/"isSufficient"/"quality"等主观自评字段
  - 质量判断的职责在主 Agent，不在 Knowledge（P6）
- 需求来源：架构.md §5.1 + §8.1；原则 P6
