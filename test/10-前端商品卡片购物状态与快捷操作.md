## FESHOP-001 快捷操作列表完整展示
- 模块：前端快捷操作
- 用例类型：正向
- 优先级：高
- 前置条件：页面初始加载完成
- 测试步骤：
  1. 查看快捷操作区域。
- 预期结果：
  - 至少展示商品咨询、游戏手机推荐、加购演示、查库存、查物流、清除记忆快捷入口。
  - 快捷操作文案清晰，不依赖图标理解含义。
- 需求来源：前端UIUX需求文档 §4.2；前端UIUX设计文档 §5.3

## FESHOP-002 点击快捷操作填入输入框
- 模块：前端快捷操作
- 用例类型：正向

## FESHOP-003 快捷操作触控面积符合要求

- 用例类型：边界
- 优先级：中

## FESHOP-004 商品推荐卡片空状态
- 模块：商品推荐卡片
- 用例类型：正向
- 优先级：中
- 前置条件：尚未触发推荐/咨询
- 测试步骤：

## FESHOP-005 推荐场景展示商品卡片
- 模块：商品推荐卡片
- 用例类型：正向
- 优先级：高
- 前置条件：`/api/chat` 返回游戏手机推荐类 answer
- 测试步骤：
  1. 发送“帮我推荐一款适合打游戏的手机”。
  2. 查看左侧推荐卡片区。
- 预期结果：
  - 展示至少 1 张商品推荐卡片。
  - 卡片包含商品名、核心标签、规格、推荐理由、操作按钮。
  - Agent 原始 answer 仍在聊天区展示。
- 需求来源：前端UIUX需求文档 §4.3；前端技术架构文档 §11.4

## FESHOP-006 商品卡片查库存按钮行为
- 模块：商品推荐卡片 / 操作按钮
- 用例类型：正向
- 优先级：中
- 前置条件：存在商品卡片，商品含 skuId
- 测试步骤：
  1. 点击商品卡片“查库存”。
- 预期结果：
  - 输入框填入或发送“查一下 <skuId> 有没有库存”类自然语言。
  - 请求仍走 `/api/chat`。
  - 不直接访问 MCP Server。
- 需求来源：前端UIUX设计文档 §8.3；前端技术架构文档 §11.4

## FESHOP-008 加购成功更新购物车状态卡
- 模块：购物车状态卡
- 用例类型：正向
- 优先级：高
- 前置条件：`/api/chat` 返回 `intent=TOOL,answer` 含加购成功和 cartId
- 测试步骤：
  1. 发送加购演示 prompt。
  2. 等待成功响应。
  3. 查看购物车状态区。
- 预期结果：
  - 聊天区展示后端 answer。
  - 购物车状态卡展示成功态。
  - 展示商品/规格/数量/cartId 中可识别的信息。
- 需求来源：前端UIUX需求文档 §4.4；前端技术架构文档 §11.4

## FESHOP-011 购物工具缺槽位显示需补充状态
- 模块：购物车状态卡 / 澄清
- 用例类型：异常
- 优先级：高
- 前置条件：`/api/chat` 返回 missing slot 文案，例如缺 `spec` 或 `address`
- 测试步骤：
  1. 触发缺规格加购或缺地址下单。
  2. 查看购物车状态卡。
- 预期结果：
  - 状态卡显示黄色需补充态。
  - 展示缺失项中文名称。
  - 不显示成功态。
- 需求来源：前端UIUX设计文档 §5.4 / §9.3

## FESHOP-012 轻量解析失败时仍可读
- 模块：responseParser / 兜底
- 用例类型：边界
- 优先级：中
- 前置条件：`/api/chat.answer` 格式无法解析为卡片数据
- 测试步骤：
  1. mock 返回非结构化 answer。
  2. 查看聊天区和卡片区。
- 预期结果：
  - 聊天区完整展示 answer。
  - 卡片区不报错、不白屏。
  - 可显示“暂无可结构化展示的结果”或保持原状态。
- 需求来源：前端技术架构文档 §11.4
