# 11-前端后端状态面板与降级错误状态

## FESTATUS-001 页面初始化自动拉取后端状态
- 模块：后端状态面板
- 用例类型：正向
- 优先级：高
- 前置条件：`/api/health` 和 `/api/ready` mock 正常返回
- 测试步骤：
  1. 打开前端页面。
  2. 观察网络请求和状态面板。
- 预期结果：
  - 页面初始化时调用 `/api/health`。
  - 页面初始化时调用 `/api/ready`。
  - 状态面板展示聚合状态和核心模块状态。
- 需求来源：前端UIUX需求文档 §4.5；接口文档 §4

## FESTATUS-002 UP 状态展示为正常
- 模块：后端状态面板 / 状态映射
- 用例类型：正向
- 优先级：高
- 前置条件：`/api/ready.status=UP` 且核心项均为 UP/CONFIGURED
- 测试步骤：
  1. mock ready 全部正常。
  2. 查看 TopBar 和状态面板。
- 预期结果：
  - TopBar 显示绿色“正常”或 `UP`。
  - 各项状态芯片为绿色语义。
  - 不显示降级/异常警告。
- 需求来源：前端UIUX设计文档 §5.4；接口文档 §4.4

## FESTATUS-003 DEGRADED 状态展示降级运行
- 模块：后端状态面板 / 降级
- 用例类型：异常
- 优先级：高
- 前置条件：`/api/ready.status=DEGRADED`，`rerank=FALLBACK`
- 测试步骤：
  1. mock ready 返回 DEGRADED。
  2. 查看 TopBar 和状态面板。
- 预期结果：
  - TopBar 显示黄色“降级运行”。
  - 状态面板突出展示具体降级项。
  - 聊天入口仍可使用，但调试区域提示部分能力降级。
- 需求来源：前端UIUX设计文档 §9.3；接口文档 §4.5

## FESTATUS-004 DOWN 状态展示核心异常
- 模块：后端状态面板 / 异常
- 用例类型：异常
- 优先级：高
- 前置条件：`/api/ready.status=DOWN` 或核心 Bean 状态为 DOWN
- 测试步骤：
  1. mock ready 返回 DOWN。
  2. 查看页面状态。
- 预期结果：
  - TopBar 显示红色异常状态。
  - 状态面板列出 DOWN 项。
  - 页面给出“后端核心服务未就绪”提示。
  - 是否禁用发送按钮按技术架构配置执行，但必须有明确提示。
- 需求来源：前端技术架构文档 §12.1；接口文档 §4.5

## FESTATUS-005 MISSING_KEY 状态展示模型配置缺失
- 模块：后端状态面板 / 模型配置
- 用例类型：异常
- 优先级：中
- 前置条件：`chatModel=MISSING_KEY` 或 `embeddingModel=MISSING_KEY`
- 测试步骤：
  1. mock ready 返回模型 key 缺失。
  2. 查看模型能力分组。
- 预期结果：
  - 缺失项显示红色或异常标识。
  - 文案说明“模型配置缺失，AI 能力可能不可用”。
  - 不只通过颜色表达异常。
- 需求来源：前端UIUX设计文档 §9.2；接口文档 §4.4

## FESTATUS-006 MCP Server DOWN 提示购物工具不可用
- 模块：后端状态面板 / MCP
- 用例类型：异常
- 优先级：中
- 前置条件：`mcpserver=DOWN`
- 测试步骤：
  1. mock ready 返回 mcpserver DOWN。
  2. 查看状态面板。
- 预期结果：
  - MCP Server 状态红色异常。
  - 文案提示加购/下单能力可能受影响。
  - 知识问答能力不被错误标记为不可用。
- 需求来源：前端UIUX需求文档 §4.5；前端UIUX设计文档 §9.2

## FESTATUS-007 手动刷新状态
- 模块：后端状态面板 / 刷新
- 用例类型：正向
- 优先级：中
- 前置条件：状态面板已展示旧状态
- 测试步骤：
  1. 点击刷新按钮。
  2. 等待请求完成。
- 预期结果：
  - 刷新按钮显示 loading。
  - 重新调用 `/api/health` 和 `/api/ready` 或按实现约定刷新 ready。
  - 更新上次刷新时间。
  - loading 结束后按钮恢复可用。
- 需求来源：前端UIUX设计文档 §5.4；前端技术架构文档 §9.3

## FESTATUS-008 状态接口失败显示错误恢复
- 模块：后端状态面板 / 错误处理
- 用例类型：异常
- 优先级：高
- 前置条件：`/api/ready` 网络失败或 500
- 测试步骤：
  1. 打开页面或点击刷新。
  2. mock ready 请求失败。
- 预期结果：
  - 状态面板显示错误文案。
  - 提供重试按钮。
  - 页面不白屏。
  - 聊天区域按当前可用状态独立显示。
- 需求来源：前端UIUX设计文档 §9.2；前端技术架构文档 §12.1

## FESTATUS-009 状态面板折叠与展开
- 模块：后端状态面板 / 信息密度
- 用例类型：正向
- 优先级：低
- 前置条件：状态数据已加载
- 测试步骤：
  1. 查看默认状态面板。
  2. 点击展开详情。
  3. 点击收起。
- 预期结果：
  - 默认展示聚合状态和核心模块。
  - 展开后展示基础设施和模型能力详情。
  - 收起后信息密度降低，不影响聊天主流程。
- 需求来源：前端UIUX设计文档 §5.4

## FESTATUS-010 状态颜色不作为唯一信息载体
- 模块：后端状态面板 / 无障碍
- 用例类型：边界
- 优先级：高
- 前置条件：ready 同时存在 UP、DEGRADED、DOWN 项
- 测试步骤：
  1. 查看状态面板。
- 预期结果：
  - 每个状态除了颜色，还包含文本或图标标签。
  - 色弱用户仍能区分正常、降级、异常。
- 需求来源：前端UIUX设计文档 §11.1
