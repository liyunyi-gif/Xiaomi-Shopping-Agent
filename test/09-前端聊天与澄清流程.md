# 09-前端聊天与澄清流程

## FECHAT-001 聊天页面初始欢迎态
- 模块：前端聊天 / 初始态
- 用例类型：正向
- 优先级：高
- 前置条件：前端应用启动，尚未发送任何消息
- 测试步骤：
  1. 打开前端首页。
  2. 查看聊天主区域。
- 预期结果：
  - 展示欢迎文案：“你好，我是小米智能导购...”或等价内容。
  - 输入框可见且可输入。
  - 快捷操作入口可见。
  - 不出现错误态或空白主区域。
- 需求来源：前端UIUX需求文档 §4.1 / §5；前端UIUX设计文档 §9.4

## FECHAT-002 用户发送商品咨询消息
- 模块：前端聊天 / 消息发送
- 用例类型：正向
- 优先级：高
- 前置条件：`POST /api/chat` mock 返回 `intent=KNOWLEDGE`
- 测试步骤：
  1. 在输入框输入“小米14的影像规格怎么样？”。
  2. 点击发送按钮。
- 预期结果：
  - 用户消息立即出现在消息列表右侧。
  - 发送期间出现 Agent loading bubble。
  - 接口返回后展示 Agent 回复。
  - Chat Header 展示 `KNOWLEDGE` 或“知识问答”。
- 需求来源：前端UIUX需求文档 §4.1；接口文档 §5

## FECHAT-003 空输入不可发送
- 模块：前端聊天 / 输入校验
- 用例类型：边界
- 优先级：高
- 前置条件：输入框为空或仅包含空格
- 测试步骤：
  1. 输入空格。
  2. 点击发送按钮或按 Enter。
- 预期结果：
  - 不调用 `/api/chat`。
  - 不新增用户消息。
  - 发送按钮保持禁用或无效状态。
- 需求来源：前端技术架构文档 §8 / §10.2 / §11.1

## FECHAT-004 Enter 发送与 Shift+Enter 换行
- 模块：前端聊天 / 输入交互
- 用例类型：正向
- 优先级：中
- 前置条件：输入框获得焦点
- 测试步骤：
  1. 输入第一行文本。
  2. 按 Shift+Enter。
  3. 输入第二行文本。
  4. 按 Enter。
- 预期结果：
  - Shift+Enter 在输入框内换行。
  - Enter 触发发送。
  - 发送内容保留换行语义或按设计规范归一化。
- 需求来源：前端UIUX设计文档 §8.6；前端技术架构文档 §10.2

## FECHAT-005 请求中禁用重复发送
- 模块：前端聊天 / loading
- 用例类型：异常
- 优先级：高
- 前置条件：`/api/chat` 响应延迟 2 秒
- 测试步骤：
  1. 输入并发送一条消息。
  2. loading 期间连续点击发送按钮。
- 预期结果：
  - loading 期间发送按钮禁用或显示 spinner。
  - 只发起 1 次 `/api/chat` 请求。
  - 不产生重复用户消息。
- 需求来源：前端UIUX设计文档 §9.1 / §11.2；前端技术架构文档 §11.1

## FECHAT-006 Knowledge 回复质量信息展示
- 模块：前端聊天 / 元信息展示
- 用例类型：正向
- 优先级：中
- 前置条件：`/api/chat` 返回 `qualityLevel=SUFFICIENT,retryCount=0,childCalls=1`
- 测试步骤：
  1. 发送知识问答消息。
  2. 等待响应。
- 预期结果：
  - Agent 回复展示 `answer`。
  - 调试信息区域展示意图、质量等级、重检次数、子节点调用次数。
  - 调试信息不干扰主阅读。
- 需求来源：接口文档 §5.5；前端UIUX设计文档 §5.2

## FECHAT-007 资料不足时展示澄清提示
- 模块：前端聊天 / 检索退化
- 用例类型：异常
- 优先级：高
- 前置条件：`/api/chat` 返回 `intent=KNOWLEDGE,needClarify=true,qualityLevel=FAILED`
- 测试步骤：
  1. 发送“这个拍照怎么样？”。
  2. 等待响应。
- 预期结果：
  - Agent 气泡展示资料不足/请补充型号的文案。
  - 气泡带“需要补充信息”或等价状态标识。
  - 输入框 placeholder 引导用户补充具体型号或参数。
- 需求来源：接口文档 §6.2；前端UIUX需求文档 §5.2

## FECHAT-008 Shopping 缺槽位澄清流程
- 模块：前端聊天 / 槽位澄清
- 用例类型：正向
- 优先级：高
- 前置条件：`/api/chat` 返回 `intent=TOOL,needClarify=true,answer=还需要补充[商品规格]...`
- 测试步骤：
  1. 发送“帮我加购小米14”。
  2. 查看 Agent 回复。
  3. 继续输入“16GB+512GB”。
- 预期结果：
  - 第 1 次响应展示澄清气泡。
  - 第 2 次请求沿用同一 `conversationId`。
  - 前端不直接调用 Shopping 或 MCP Server。
- 需求来源：前端UIUX需求文档 §5.2；接口文档 §5.8 / §6.5

## FECHAT-009 多轮会话保持 conversationId
- 模块：前端聊天 / 会话状态
- 用例类型：正向
- 优先级：高
- 前置条件：已生成 `conversationId`
- 测试步骤：
  1. 连续发送 3 条消息。
  2. 检查每次 `/api/chat` 请求体。
- 预期结果：
  - 3 次请求使用同一个 `conversationId`。
  - 页面刷新前同一会话消息连续展示。
- 需求来源：前端UIUX需求文档 §4.1 / §8.2；前端技术架构文档 §9.2

## FECHAT-010 网络错误显示恢复路径
- 模块：前端聊天 / 错误处理
- 用例类型：异常
- 优先级：高
- 前置条件：`/api/chat` mock 网络失败或超时
- 测试步骤：
  1. 发送任意消息。
  2. 等待请求失败。
- 预期结果：
  - loading 状态结束。
  - 展示 error bubble。
  - 错误文案说明原因并提供重试或重新发送路径。
  - 不丢失用户原始输入语义。
- 需求来源：前端UIUX设计文档 §9.2；前端技术架构文档 §12.1

## FECHAT-011 系统指令回复展示
- 模块：前端聊天 / 系统指令
- 用例类型：正向
- 优先级：中
- 前置条件：`/api/chat` 返回 `intent=SYSTEM`
- 测试步骤：
  1. 发送“清除我的记忆”。
  2. 等待响应。
- 预期结果：
  - Agent 回复以系统消息或中性提示样式展示。
  - `childCalls=0` 的调试信息可正确展示。
- 需求来源：接口文档 §6.10；前端UIUX需求文档 §4.2

## FECHAT-012 消息列表自动滚动
- 模块：前端聊天 / 消息列表
- 用例类型：正向
- 优先级：中
- 前置条件：消息列表已超过一屏
- 测试步骤：
  1. 连续发送多条消息。
  2. 等待最后一条 Agent 回复。
- 预期结果：
  - 新消息出现后列表滚动到最新消息。
  - 用户手动向上查看历史时，不应强制频繁打断阅读（可按实现策略保留底部新消息提示）。
- 需求来源：前端UIUX设计文档 §5.2；前端技术架构文档 §10.2
