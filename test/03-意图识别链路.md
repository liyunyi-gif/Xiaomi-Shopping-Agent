# 测试用例 · 意图识别链路

> 模块：Orchestrator 主 Agent · intent
> 前缀：`INTENT`
> 覆盖验收点：架构.md §3.1（意图识别唯一入口、三类一级意图、澄清权）+ 原则 P1、P2

---

## 验收目标

1. **P1 意图识别唯一入口**：只有主 Agent 做意图识别，子节点不做。
2. 三类一级意图正确分类：KNOWLEDGE（知识库问答）/ TOOL（工具调用）/ SYSTEM（系统指令）。
3. 低置信度触发澄清反问（主 Agent 唯一开口权 P2）。
4. 实体抽取（型号/参数）供后续质量判断使用。
5. 二级意图归类（商品推荐/参数咨询/加购/下单/物流等）。

---

## INTENT-001 意图识别唯一入口（约束）★ P1

- 模块：意图识别·入口唯一性
- 用例类型：正向
- 优先级：高
- 前置条件：任意用户 query
- 测试步骤：
  1. 审查 Knowledge / Shopping 子节点的代码与接口
  2. 注入任意 query 给子节点
- 预期结果：
  - 子节点接口**不接收**原始 query 做意图判断
  - 子节点只接收"已明确的意图 + 快照"（KnowledgeRequest.intent / ShoppingRequest.action 已是确定值）
  - 意图识别逻辑仅存在于主 Agent
- 需求来源：架构.md §3.1；原则 P1（不可变）

---

## INTENT-002 知识库问答意图分类（正向）

- 模块：意图识别·一级意图
- 用例类型：正向
- 优先级：高
- 前置条件：query="小米14的影像规格怎么样"
- 测试步骤：
  1. 主 Agent 意图识别
- 预期结果：
  - primaryIntent = KNOWLEDGE
  - 二级意图 = 参数咨询/规格咨询
  - 实体抽取命中 {"小米14", "影像"}
- 需求来源：架构.md §3.1

---

- 预期结果：

  - 二级意图 = 加购
  - 实体抽取命中 {"小米14", "16+512"}

## INTENT-004 系统指令意图分类（正向）

- 模块：意图识别·一级意图
- 用例类型：正向
- 优先级：中
- 前置条件：query="清除我的记忆" / "查看历史对话"
- 测试步骤：
  1. 主 Agent 意图识别
- 预期结果：
  - primaryIntent = SYSTEM
  - 二级意图 = 记忆清除 / 历史回显
  - 该类由主 Agent 自处理，不委派子节点
- 需求来源：架构.md §3.1

---

## INTENT-005 低置信度触发澄清反问（异常/边界）★ P2

- 模块：意图识别·澄清权
- 用例类型：边界
- 优先级：高
- 前置条件：query="手机"（意图模糊，无法判断是想了解还是想买）
- 测试步骤：
  1. 主 Agent 意图识别
  2. 置信度低于阈值（如 < 0.6）
- 预期结果：
  - needClarify = true
  - 主 Agent 生成意图级澄清反问（"您是想了解手机信息，还是想购买？"）
  - **由主 Agent 开口**（唯一开口权 P2），不委派子节点
  - 在澄清前不调用任何子节点
- 需求来源：架构.md §3.1（澄清权）；原则 P2

---

## INTENT-006 槽位缺失触发槽位澄清（异常）

- 模块：意图识别·槽位澄清
- 用例类型：异常
- 优先级：高
- 前置条件：query="帮我下单"（意图明确=下单，但缺商品/数量/地址槽位）
- 测试步骤：
  1. 主 Agent 识别意图=下单（高置信）
  2. 检测槽位缺失
- 预期结果：
  - 意图识别成功（不触发意图澄清）
  - 但触发**槽位澄清**：主 Agent 开口询问商品/数量/收货地址
  - 注：该问哪些槽位可由 Shopping 声明（举手），但**开口永远是主 Agent**
- 需求来源：架构.md §3.1（槽位澄清）；原则 P4
